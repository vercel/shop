The template includes opt-in customer authentication using the framework-neutral session and OAuth helpers from `@shopify/hydrogen/customer-account`. Customers can sign in through Shopify, view their profile, order history, and address book, refresh expired access tokens, and sign out of both the storefront and Shopify identity provider.

Authentication is disabled by default. The canonical default lives in `shop.config.ts`, and a deployment can override it with `NEXT_PUBLIC_ENABLE_AUTH`. When disabled, the storefront works as a guest-only experience and no auth UI is rendered.

## Configuration

Set the feature flag, Customer Account API client ID, session secret, and public storefront URL:

```bash
NEXT_PUBLIC_ENABLE_AUTH="1"
CUSTOMER_ACCOUNT_SESSION_SECRET="[redacted]"
SHOPIFY_CUSTOMER_ACCOUNT_API_CLIENT_ID="your-customer-account-client-id"
NEXT_PUBLIC_BASE_URL="https://your-domain.com"
```

Shopify provides `SHOPIFY_CUSTOMER_ACCOUNT_API_CLIENT_ID`, but it does not provide `CUSTOMER_ACCOUNT_SESSION_SECRET`. Generate that private, app-owned secret with `openssl rand -base64 32`. The template uses it to derive the AES-256-GCM key that encrypts Hydrogen's OAuth session data in HttpOnly cookies. Keep the value server-only and consistent across deployment instances. Rotating it invalidates every existing customer session.

The session secret is required by this template's encrypted-cookie storage, not by the Customer Account API itself. If you replace the cookie implementation with opaque session IDs backed by protected server-side storage, use that system's key management instead. Do not remove the secret while storing access, refresh, and ID tokens in client-held cookies.

`next.config.ts` throws at build time whenever auth is enabled without the session secret and client ID. The `NEXT_PUBLIC_ENABLE_AUTH` override must remain public so conditional rendering matches between server and client under cache components.

Customer Account OAuth requires a public HTTPS origin. Use a tunnel for local OAuth testing and set `NEXT_PUBLIC_BASE_URL` to that registered origin.

### Shopify Admin setup

1. Enable customer accounts: **Shopify Admin → Settings → Customer accounts → Edit**, choose **Customer accounts**, and Save
2. Install the **Headless** sales channel from the Shopify App Store
3. Go to **Sales channels → Headless → (your storefront) → Customer Account API**
4. Use a public Customer Account API client; Hydrogen performs authorization code + PKCE without a client secret
5. Set the callback URI to `{YOUR_DOMAIN}/account/authorize`
6. Set the logout URI to `{YOUR_DOMAIN}/`
7. Copy the client ID to `SHOPIFY_CUSTOMER_ACCOUNT_API_CLIENT_ID`
8. Ensure the store domain matches `SHOPIFY_STORE_DOMAIN`

Shopify does not support wildcard callback or logout URIs. Register each production, preview, or tunnel origin that needs authentication.

See [Environment Variables](/docs/reference/env-vars) for the full variable reference.

## How it works

Authentication is built on these modules:

| Module                            | Purpose                                                                                                                                                                                   |
| --------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `lib/auth/index.ts`               | Universal `isAuthEnabled` flag. Safe to import from server and client code.                                                                                                               |
| `lib/auth/server.ts`              | Encrypted, chunked HttpOnly cookie storage plus Hydrogen session helpers for login state, strict access-token reads, refresh redirects, and request-scoped session managers. Server-only. |
| `app/account/[action]/route.ts`   | Installs Hydrogen's standard login, authorize, refresh, and logout handlers through `handleShopifyRoutes`.                                                                                |
| `lib/shopify/customer-account.ts` | Creates Hydrogen's Customer Account GraphQL client and sends the server-only access token with each operation.                                                                            |

There is no client-side auth SDK. Sign-in is a normal GET navigation to `/account/login`; sign-out is a same-origin POST form to `/account/logout`.

### Feature gating

The `isAuthEnabled` flag reads the resolved `auth.enabled` value from `lib/config`. That value uses `NEXT_PUBLIC_ENABLE_AUTH` when set and otherwise falls back to `shopConfig.auth.enabledByDefault`. When false:

- The account icon does not appear in the nav
- The Hydrogen account auth routes return 404
- Protected account pages return 404
- No auth-related code runs at request time

## Routes

| Route                  | Method | Description                                                                                                   |
| ---------------------- | ------ | ------------------------------------------------------------------------------------------------------------- |
| `/account/login`       | GET    | Creates PKCE state in the encrypted session cookie and redirects to Shopify. Accepts sanitized `return_to`.   |
| `/account/authorize`   | GET    | Validates the OAuth callback, exchanges the code, stores tokens, and redirects to the original account route. |
| `/account/refresh`     | GET    | Refreshes an expired access token at a writable response boundary and redirects back to `return_to`.          |
| `/account/logout`      | POST   | Clears the local session and redirects through Shopify's logout endpoint before returning home.               |
| `/account`             | GET    | Redirects to `/account/profile`.                                                                              |
| `/account/profile`     | GET    | View and edit the customer's name; email shown read-only.                                                     |
| `/account/orders`      | GET    | Paginated order history, newest first.                                                                        |
| `/account/orders/[id]` | GET    | Order detail: line items, totals, shipping address, and status-page link.                                     |
| `/account/addresses`   | GET    | Address book with create, edit, delete, and default selection.                                                |

The `(authenticated)` route group gates protected account pages without claiming the auth-handler paths.

## Architecture

### Session flow

1. `/account/login` asks Hydrogen to generate state, nonce, and PKCE values
2. The app encrypts Hydrogen's pending-login state into chunked AES-256-GCM HttpOnly cookies
3. Shopify redirects to `/account/authorize`
4. Hydrogen validates state, nonce, issuer, audience, and ID-token expiry before exchanging the authorization code
5. Access, refresh, and ID tokens replace the pending-login state in the encrypted cookie
6. Server Components only perform read-only login and token checks; they never mutate cookies

### Token refresh

`isCustomerLoggedIn()` treats a refreshable session as logged in, so the nav does not flicker to signed out when the access token expires. Customer Account API operations call `requireCustomerAccessToken()`. If only a refresh token remains, it redirects through `/account/refresh`, where Hydrogen can rotate tokens and commit the updated cookie before returning to the account page.

### Logout

The sign-out button submits a same-origin POST to `/account/logout`. Hydrogen clears the local session, includes the stored ID token as `id_token_hint`, and redirects through Shopify's logout endpoint. This ends both the storefront session and Shopify SSO session before returning to `/`.

### Server-side usage

```ts
import {
  isCustomerLoggedIn,
  requireCustomerAccessToken,
  requireCustomerSession,
} from "@/lib/auth/server";

const loggedIn = await isCustomerLoggedIn();
await requireCustomerSession();
const accessToken = await requireCustomerAccessToken("/account/orders");
```

Use `isCustomerLoggedIn()` for read-only UI state, `requireCustomerSession()` for account-page gates, and `requireCustomerAccessToken()` immediately before private Customer Account API work.

### Account data

The account pages read and write customer data through the Shopify Customer Account API, a separate GraphQL endpoint and schema from the Storefront API.

| Module                               | Purpose                                                                                                                                                                       |
| ------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `lib/shopify/customer-account.ts`    | `customerAccountFetch()` wraps Hydrogen's `createCustomerAccountClient`. The numeric shop ID is derived from the Shopify OIDC discovery issuer and cached.                    |
| `lib/shopify/operations/customer.ts` | Queries profile, orders, order detail, and addresses; mutates addresses and profile data. Each call resolves a currently usable token through `requireCustomerAccessToken()`. |
| `lib/shopify/transforms/customer.ts` | Maps Customer Account API responses to provider-agnostic domain types in `lib/types.ts`.                                                                                      |
| `lib/customer/action.ts`             | Server actions for address and profile forms. They validate input, surface Shopify `userErrors`, and revalidate affected paths.                                               |

Order and profile pages are Server Components wrapped in Suspense. Address and profile editing use client forms that call server actions. Customer responses are personalized and never publicly cached.

## Guardrails

- Never expose access, refresh, or ID tokens to Client Components or browser-readable storage
- Keep Server Components on the read-only session-manager path; token refresh belongs in `/account/refresh`
- Always resolve a current access token before a Customer Account API operation
- Keep login as GET and logout as same-origin POST
- Keep the callback and logout URIs registered exactly in Shopify
- Preserve Hydrogen's PKCE, state, nonce, return-path sanitization, and `no-store` response behavior
- Rotate `CUSTOMER_ACCOUNT_SESSION_SECRET` deliberately; rotation invalidates every existing encrypted session
- Keep `NEXT_PUBLIC_ENABLE_AUTH` public so server/client feature gates agree under cache components

## Extending

To add account data such as store credit, subscriptions, draft orders, or richer order fields, add an operation in `lib/shopify/operations/customer.ts`, a transform in `lib/shopify/transforms/customer.ts`, and a matching domain type in `lib/types.ts`. Validate new fields against the live Customer Account API schema. Mutations should run through a server action in `lib/customer/action.ts` so validation and revalidation stay centralized.
