> **Important:** The template expects three menus in your commerce admin with exact handles: `main-menu`, `footer`, and `quick-links`. If a menu doesn't exist, the corresponding section renders empty.

The navigation system is split into a sticky top navbar for desktop and a fixed bottom bar for mobile. Both share a megamenu component that renders menu data from the commerce provider as a multi-level category browser.

## Navbar

A sticky header (`z-30`) that composes the logo/brand link, megamenu trigger, quick links, an about link (hidden on mobile), and the cart icon. On mobile, only the logo and cart icon are visible - the rest of the navigation moves to the bottom bar.

## Megamenu

The megamenu has a server component that fetches menu data via `getMenu("main-menu", locale)` and a client component that handles interaction.

The menu supports a 3-level hierarchy: top-level items, panels, and categories. On desktop, panels activate on hover with mouse direction tracking - the close delay increases when the cursor moves toward the open panel, preventing accidental dismissal. On mobile, the menu renders as a full-screen overlay with keyboard escape support.

Communication between the mobile bottom bar's browse button and the megamenu uses the `BroadcastChannel` API, allowing the two independent component trees to coordinate open/close state without shared React context.

## Quick links

A secondary horizontal link bar sourced from the commerce menu with ID `"quick-links"`. Supports both internal and external URLs. Hidden on mobile (`md:flex`).

## Cart icon

The cart icon is a server/client component pair. The server component reads the `cartId` cookie and fetches the initial cart. The client component renders a quantity badge with an animation for pending additions and opens the cart overlay on click.

## Mobile bottom bar

A fixed bar at the bottom of the viewport, visible below the `md` breakpoint. It contains:

- **Browse button** - toggles the megamenu via `BroadcastChannel`
- **Search button** - expands an animated search input
- **Agent button** - optional slot, rendered when enabled

Transitions between states use Framer Motion for smooth animation.

## Responsive behavior

**Desktop** (`md` and up) - full horizontal navbar with megamenu hover interaction, quick links, about link, and cart sheet.

**Mobile** (below `md`) - minimal top bar with logo and cart icon. Primary navigation moves to the fixed bottom bar with browse, search, and agent buttons.
