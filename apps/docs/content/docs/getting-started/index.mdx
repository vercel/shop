## Set up your commerce provider

If you don't already have a commerce provider account, create one with your provider of choice. Then configure API access:

1. In your provider's admin panel, navigate to the API or developer settings
2. Create a new storefront or headless channel
3. Copy the **API access token** — you'll need this for your environment variables
4. Enable the API permissions listed below

### Required API permissions

Your commerce provider needs to expose the following capabilities:

| Capability | Purpose |
|---|---|
| Read products | Product pages, search, recommendations |
| Read inventory | Stock availability on PDPs and PLPs |
| Read collections | Collection pages and navigation |
| Write cart / checkout | Cart operations and checkout |
| Read content / pages | CMS pages and content objects |

Additional permissions are needed if you enable optional features:

| Capability | Required by |
|---|---|
| Read customer data | [Enable Auth](/docs/skills/enable-auth) skill |
| Read content objects | [Enable CMS](/docs/skills/enable-cms) skill |

## Create your project

There are three ways to get started:

### Deploy with Vercel

The fastest way — deploy directly to Vercel and it will create the repository for you.

</a>

### Clone with create-next-app

```bash
npx create-next-app@latest --example vercel/shop --example-path apps/template
```

### Start from v0

You can also use [v0](https://v0.dev) to generate a storefront and iterate on the design before connecting it to your commerce provider.

## Environment variables

In your provider's admin panel, locate the API credentials for your storefront. Create a `.env.local` file in your project root:

```bash
COMMERCE_STORE_DOMAIN="your-store.example.com"
COMMERCE_API_TOKEN="your-api-access-token"
NEXT_PUBLIC_SITE_NAME="Your Store Name"
```

## Run locally

```bash
pnpm install
pnpm dev
```

Your store is now running at [localhost:3000](http://localhost:3000).

## AI coding agents

If you use [Claude Code](https://claude.ai/code) or [Cursor](https://cursor.sh), install the [Vercel plugin](https://vercel.com/changelog/introducing-vercel-plugin-for-coding-agents) to get platform-aware guidance for Next.js, AI SDK, shadcn/ui, caching, and more — 47+ skills that activate automatically as you work.

### Claude Code

```bash
npx plugins add vercel/vercel-plugin
```

### Cursor

```bash
/add-plugin vercel
```

The plugin provides real-time context about Vercel APIs, catches deprecated patterns, and includes slash commands like `/bootstrap`, `/deploy`, `/env`, `/status`, and `/marketplace`.

## Deploy

Push your repository to GitHub and import it on [vercel.com/new](https://vercel.com/new). Add the same environment variables from your `.env.local` to your Vercel project settings.
