## Products not showing up

The most common cause is missing API permissions. In your store admin, go to your storefront app settings and verify these scopes are enabled:

| Scope | Required for |
|---|---|
| `unauthenticated_read_product_listings` | Product pages and search |
| `unauthenticated_read_product_inventory` | Stock availability |
| `unauthenticated_read_collection_listings` | Collection pages |
| `unauthenticated_write_checkouts` | Cart and checkout |
| `unauthenticated_read_content` | CMS pages |

Also check that your products are set to the appropriate sales channels in your store admin.

## Cart not persisting across page loads

The cart ID is stored in a `cartId` HTTP-only cookie with a 7-day expiry. Common causes:

- **Cookie domain mismatch** - if you're running on a different domain than expected, the cookie may not be sent. Check your browser dev tools → Application → Cookies.
- **Missing `invalidateCartCache()`** - every cart mutation must call `invalidateCartCache()` from `@/lib/cart-cache`. Without it, the cached cart data goes stale and the UI may show an outdated state.

## Variant selection not working

Variant selection relies on a rewrite rule in `next.config.ts` that maps `?variantId=123` to a route segment:

```ts
rewrites: async () => [
  {
    source: "/products/:handle",
    has: [{ type: "query", key: "variantId", value: "(?<variantId>.+)" }],
    destination: "/products/:handle/:variantId",
  },
],
```

If this rule is missing or modified, clicking a variant option will append a query parameter but the page won't re-render with the new variant.

## Images not loading

Your commerce provider serves images from its CDN. The template's `next.config.ts` must include the provider's image domain in `images.remotePatterns`. Verify that your provider's CDN hostname is listed:

```ts
images: {
  remotePatterns: [
    { hostname: "your-provider-cdn.example.com" },
  ],
},
```

## Build fails with GraphQL errors

The template includes a local API schema snapshot in `.claude/schemas/`. If the schema is outdated relative to the API version in use, queries may reference fields that don't exist.

Refresh the schema by running the fetch script in `.claude/scripts/`.

## Changes in your store not appearing on the site

All product and collection data is cached with `cacheLife("max")`. Without webhooks configured, changes only appear when the cache expires.

To get near-instant updates, set up webhooks pointed at `/api/webhooks`. See the webhooks and cache invalidation guide for the full setup.

## Locale or currency not changing

Multi-locale support requires Markets to be enabled. The template ships as single-locale by default. Run the [Enable Markets](/docs/skills/enable-markets) skill to add multi-locale and multi-currency routing.

## Agent can't find context files

Coding agents rely on `AGENTS.md` in the project root for architecture context and `.agents/skills/` for task playbooks. If these files are missing:

- Verify you cloned from the correct template (`vercel/shop`, `apps/template` path)
- Check that `.agents/` isn't in your `.gitignore`
- The `AGENTS.md` file should be at the project root, not nested in a subdirectory
