This page covers the Shopify admin configuration the PDP expects. For how the page renders and how variant selection works, see [Anatomy → PDP](/docs/anatomy/pages/pdp).

## Basic setup

A product needs a **title** and **description** (rendered as HTML via `descriptionHtml`), at least one **variant** (single-option products still get a default variant), and a **featured image** plus up to 10 media items. The PDP shell fetches options and Shopify's encoded variant-availability data rather than the full variant list, resolving the selected variant per request from its options; the AI agent and markdown routes fetch the full variant list (up to 250) via `getProductWithVariants`.

Products must belong to at least one collection to appear on collection listing pages.

## Color swatches

A product option with swatch data (a hex color or swatch image, read from `optionValues.swatch`) renders as color swatches instead of text buttons. Configure it in Shopify admin: edit a product → open an option like "Color" → assign a swatch color or image to each value. Options without swatch data fall back to standard text pickers.

## SEO fields

`seo.title` and `seo.description` populate the page `<title>`/meta description and Open Graph tags when set; otherwise the template falls back to the product title and description. Product Taxonomy category (up to 3 levels, set under a product's **Category** field) feeds structured data.

## Recommendations

- **Related products** — Shopify's automatic recommendation API (purchase history and similarity). On by default; disable with `pdp.relatedProducts.enabled: false` in `shop.config.ts`.
- **Complementary products** — the merchant-curated "Pairs Well With" list, powered by `productRecommendations(intent: COMPLEMENTARY)`. Unlike related products, these are **configured by hand** per product in the free [Shopify Search & Discovery](https://apps.shopify.com/search-and-discovery) app's **Product recommendations** settings; a product with none configured renders nothing. Disable with `pdp.complementaryProducts.enabled: false`.

## Bundles

The template renders Shopify [product bundles](https://help.shopify.com/en/manual/products/bundles) from the Storefront API's variant-level relationships: `components` (a fixed bundle's items and quantities), `groupedBy` (up to 10 bundles containing the current product), and `requiresComponents` (marks a customized bundle that needs shopper-selected components). Disable with `pdp.bundles.enabled: false`.

Fixed bundles (e.g. from the free **Shopify Bundles** app) render and add to cart normally. A customized bundle (`requiresComponents` set, no fixed `components`) stays disabled in the buy buttons until you build a component picker that sends selections through `CartLineInput.parent`.

> **Publish bundles to your Headless channel.** Shopify Bundles publishes to **Online Store** only; the template reads from **Headless**. A bundle that 404s on the storefront but looks fine in admin is almost always missing from Headless — add it under the product's **Publishing** section. See [Troubleshooting → Bundles not appearing](/docs/reference/troubleshooting).
